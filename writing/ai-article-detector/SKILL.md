# AI 文章检测 Skill

## 功能

- 输入一篇文章的 URL 链接，或直接粘贴文章内容
- 从 5 个维度打分，识别文章 AI 写作倾向
- 输出总 AI 概率 + 雷达图 + 饼图的 HTML 可视化报告

## 5 个评分维度（各 0–100）

| 维度 | 说明 |
|------|------|
| **结构维度** | 是否过度工整（小标题、过渡句、总结段每段都有） |
| **语言维度** | 是否高频出现"首先/其次/再者/总之"、"值得注意的是"、"不仅…而且…" |
| **信息密度** | 是否车轱辘话多、每段信息增量少、缺少具体案例/数据/人名/时间 |
| **情感立场** | 是否完全中立、无主观判断、无个人化经历、无观点冲突 |
| **格式细节** | 是否滥用 em dash（—）、重复性排比句、结尾必有"综上所述/总的来说" |

## 使用方法

### 方式一：URL 链接

```
请帮我检测这篇文章的 AI 占比：https://example.com/article
```

### 方式二：直接粘贴文本

```
请帮我检测以下文章的 AI 占比：
[粘贴文章内容]
```

## 执行步骤

当用户提供文章链接或文章内容时，执行以下步骤：

### Step 1：获取文章内容

- 如果是 URL，使用 WebFetch 工具抓取页面内容，提取正文文本（去掉导航栏、广告、评论等）
- 如果是直接粘贴的文本，直接使用

### Step 2：5 个维度评分

逐一分析以下维度，每个维度给出 0–100 的 AI 倾向分数和简要判断理由：

**① 结构维度（structure_score）**
- 检查是否每段都有小标题
- 是否有固定的"引言→主体→总结"三段论结构
- 是否每段都有过渡句
- AI 特征高分（>70）：结构极度工整，几乎每段都有标题+过渡+总结

**② 语言维度（language_score）**
- 统计"首先"、"其次"、"再者"、"再次"、"最后"、"总之"、"值得注意的是"、"不仅…而且…"、"与此同时"、"此外"的出现频次
- 检查是否有高度模板化的句式
- AI 特征高分（>70）：以上词汇密集出现，句式高度相似

**③ 信息密度（density_score）**
- 检查是否有具体的人名、地名、时间、数据、案例
- 每段是否有实质性新信息，还是换个说法重复上一段
- AI 特征高分（>70）：大量空泛表述，缺乏具体细节，段落间信息重复度高

**④ 情感立场（emotion_score）**
- 是否有明确的个人观点、立场偏向
- 是否有个人化经历、情绪表达、不确定性表达
- 是否存在观点冲突或自我否定
- AI 特征高分（>70）：完全中立，无任何主观判断，每个观点都"既有…也有…"

**⑤ 格式细节（format_score）**
- 统计"—"（em dash）的使用频次
- 是否有大量排比句（3 个以上并列项）
- 结尾是否有"综上所述"、"总的来说"、"总而言之"
- 是否有"数字+方面/点/步骤"的列举结构（如"三个方面"、"五个步骤"）
- AI 特征高分（>70）：以上特征密集出现

### Step 3：计算总分

总 AI 概率 = 五个维度分数的加权平均：
- 结构维度：20%
- 语言维度：25%
- 信息密度：25%
- 情感立场：20%
- 格式细节：10%

判断分类：
- 总分 ≥ 70：**高度疑似 AI**（AI 区间）
- 总分 40–69：**不确定**（混合写作，可能有 AI 辅助）
- 总分 < 40：**倾向人工**（人类区间）

### Step 4：生成 HTML 可视化报告

使用 show_widget 工具直接在对话中渲染以下内容：

1. **总体结论卡片**：AI 概率百分比 + 判断结论 + 一句话总结
2. **雷达图**：5 个维度的得分，使用 Chart.js 渲染
3. **饼图**：AI / 人类 / 不确定 的概率分布
4. **维度详情**：每个维度的得分 + 判断理由（用进度条展示）

### Step 5：给出改进建议

如果文章 AI 占比较高，给出 3 条让文章更"人味"的具体建议。

## 注意事项

- 该评分是基于文本特征的统计分析，不代表文章绝对是 AI 写的，只是 AI 倾向判断
- 部分专业领域的严谨写作也会有类似 AI 特征，需结合内容质量综合判断
- 若 URL 无法访问或内容抓取失败，提示用户直接粘贴文章内容
