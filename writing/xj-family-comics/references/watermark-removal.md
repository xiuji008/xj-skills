# AI 水印检查与去除（漫画版）

## 背景

生图服务（image_gen）常在图片底部角落叠加一行「图片由 AI 生成」类水印文字（浅灰 / 半透明 / 偶尔深色）。这与本 skill 的硬约束冲突：背景应是干净柔色，且**绝对不要**任何「图片由 AI 生成 / AI 生成」类水印或角标。

当前环境**模型无法用视觉直接看图**，且本机**未安装 Python（无 Pillow）**，仅有 PHP 8 + GD 扩展。因此水印去除走**程序化（PHP GD 底部角落填充）流程**。每张图交付前都要跑一遍，确认无残留。

> 注意：生成提示词已写「no watermark / no AI-generated mark」**并不能保证**服务不加水印（实测仍会叠加），所以必须做程序化兜底。

## 一、去除（Remove）—— 默认 SOP

水印稳定出现在**底部边缘 / 角落**，与中央绘图分离。用 PHP GD 把底部条带与两个底部角落块采样为「背景纸色」后填充覆盖即可，不会碰到中央角色与台词气泡。

脚本已随 skill 提供：`scripts/clean_watermark.php`（PHP GD，无需 Python）。用法：

```powershell
# 仅处理校准图（family-ip 下所有 png）
cd writing\xj-family-comics
php scripts/clean_watermark.php

# 处理某篇漫画目录（如 comparison-parenting-comic 下所有 png）
php scripts/clean_watermark.php assets\comparison-parenting-comic

# 处理指定文件
php scripts/clean_watermark.php a.png b.png
```

脚本逻辑（供参考，无需手改）：采样四角 + 底部中央得到背景纸色，把底部 56px 整条 + 底部左右各 320×90 角块填充为该背景色，覆盖角落水印文字，不碰中央角色与台词。

判断成功：打印的 `bg rgb` 为暖纸色（约 `250,238,217` 量级），说明角落本就是背景、填充安全未裁切角色。

## 二、兜底（水印不在底部时）

若预览发现水印出现在**顶部 / 正中**等非底部区域，再针对性处理（不要用整片清除，避免误删台词气泡）：

- 顶部角落：把脚本里的填充区域从底部（`$h - ...`）改到顶部（`0 .. 高度`）。
- 中部细文字：优先**重生成该张**（见 `SKILL.md` 工作流 3），不要试图用画笔修补。
- 若水印与中央绘图重叠难以分离，做底部填充后请用户过目确认。

## 三、交付前自检清单

- [ ] 背景为统一柔色（角落与背景一致，无灰块）。
- [ ] 无底部 / 角落「AI 生成」类水印残留（PHP 脚本已跑通）。
- [ ] 中央绘图与自带中文台词气泡完整（未被误删）。
- [ ] 三口之家形象完好：饺子大脑袋大眼、妈妈 3 个发髻、小修眼镜 + 邋遢。

## 四、限制

- 模型无法看图，结论来自像素统计；密集插图（内容铺满边缘）的水印可能和绘图重叠、无法干净分离。此时请用户过目。
- 若误删了某张图的自带台词，直接重生成该张，不要试图修补。
- 角色参考图 `assets/family-ip/family-character-reference.png` 与三张单人画像为低频校准图、不进默认生成路径；如需统一去水印，按本流程批量跑一遍即可。
