# AI 水印检查与去除

## 背景

生成服务（image_gen / 文生图接口）常常在图片上叠加一行「图片由 AI 生成」类水印文字（通常在底部边缘或某个角落，灰色或半透明，偶尔是深色）。这与本 skill 的两大硬约束冲突：

- 纯白背景（见 `style-dna.md`「必须 / 绝对不要」）。
- 绝对不要任何「图片由 AI 生成 / AI 生成」类水印文字或角标（已写入 `style-dna.md` 与 `prompt-template.md`）。

当前模型无法用视觉直接看图，因此水印的检查与去除**必须走程序化（像素分析）流程**。本文件即该流程的 SOP。

> 重要：生成提示词里写「no watermark / no AI-generated mark」**并不能保证**服务不加水印（实测仍会叠加）。所以每张图交付前都要走下面的检查。

## 一、检查（Check）

用 Pillow（无需 numpy）逐张分析。核心思路：水印是「文字状小簇」——由许多互相断开的小笔画组成（每行笔画段数多），且通常位于**底部边缘或角落**，与中央的主体绘图分离。

### 1. 背景是否为纯白

采样左上角像素 `(5,5)`。若 `min(bg) < 250`，说明是灰底，需先白化（见去除步骤 1）。

### 2. 主体绘图是否延伸到边缘

扫描底部 20% 整行的非白像素数。若数量很大（几万），说明插图本身内容铺到边缘，水印会和绘图重叠、难以分离——此时优先做背景白化 + 底部浅灰清除，再靠边缘文字簇定位。

### 3. 定位边缘「文字状小簇」（水印特征）

对四个区域分别扫描，找满足以下条件的连通簇：

- 高度 `< 70px`、宽度 `> 40px`（文字条带，不是大块绘图）
- 使用阈值 `min(r,g,b) < 200` 抽取墨迹
- 同一行内「断开的笔段数（runs）」较多（文字特征；实心形状 runs 很少）

扫描区域（按图尺寸 `w,h`）：

```text
左下 BL : x[0, 0.28w]            y[0.80h, h]
右下 BR : x[0.72w, w]            y[0.80h, h]
左上 TL : x[0, 0.20w]            y[0, 0.12h]
```

若某区域出现上述文字状小簇，且位置在极端边缘（如 `y > 0.90h` 或紧贴角落），**高度疑似水印**。跨多张图对比：服务水印常出现在相对固定的位置（或随机角落），多张图若在相似边缘出现同类小簇，基本可判定为水印。

### 4. 区分水印与自带中文标注

本 skill 的图里也会有手写中文标注（红/橙/蓝或黑），它们也是文字状小簇，容易误判。判别原则：

- **位置**：水印在极端边缘/角落；自带标注贴着它所说明的物件，多在画面中部。
- **颜色**：水印常为浅灰（接近背景）；自带标注多为深色或彩色（红/橙/蓝，`min≈0`）。
- **保守策略**：只清除极端边缘/角落的浅灰文字簇；中央区域的文字簇一律保留，避免误删标注。

## 二、去除（Remove）

### 0. 先备份

```python
import shutil, os
shutil.copy(src, src + '.bak')
```

### 1. 自适应背景白化（去灰底 + 浅灰水印）

把「接近背景灰」的像素统一刷成纯白，保留深色线稿与彩色标注：

```python
from PIL import Image
img = Image.open(path).convert('RGB'); w, h = img.size
px = img.load()
bg = px[5, 5]; bgmin = min(bg)
thr = bgmin - 10                      # 比背景略暗即视为背景
for y in range(h):
    for x in range(w):
        if min(px[x, y]) > thr:
            px[x, y] = (255, 255, 255)
```

> 浅灰水印（亮度接近背景）会被这一步一并抹掉；深色水印（`< 120`）和彩色标注不受影响。

### 2. 精准抹除边缘文字簇（深色水印）

对检查阶段定位到的边缘文字簇，按其包围盒（外扩 4px）直接填白：

```python
for (x0, x1, y0, y1) in watermark_boxes:   # 来自检查步骤的定位
    for y in range(y0 - 4, y1 + 4):
        for x in range(x0 - 4, x1 + 4):
            px[x, y] = (255, 255, 255)
```

只处理检查确认过的边缘盒子，**不要**对画面中部区域做整片清除。

### 3. 兜底：底部浅灰清除

若担心底部边缘还有未定位的浅灰水印，可只对底部 6% 区域（`y >= 0.94h`）做浅灰清除，保留深色线稿：

```python
for y in range(int(h * 0.94), h):
    for x in range(w):
        m = min(px[x, y])
        if 120 < m < 240:
            px[x, y] = (255, 255, 255)
```

### 4. 保存并复检

保存后，重新跑「检查 / 步骤 3」的定位脚本，确认目标簇已 `NONE (cleared)`。

## 三、交付前自检清单

- [ ] 背景纯白（左上角 `min≈255`）
- [ ] 无底部/角落「文字状小簇」残留（复检通过）
- [ ] 中央绘图与自带中文标注完整（未被误删）
- [ ] 饺子头小修形象、眼镜、空表情完好

## 四、限制与兜底

- 模型无法看图，结论来自像素统计；密集插图（内容铺满边缘）的水印可能和绘图重叠、无法干净分离。此时优先背景白化 + 边缘清除，并请用户过目。
- 若误删了某张图的自带标注，直接重生成该张（见 `SKILL.md` 工作流 3），不要试图修补。
- 历史资产 `assets/examples/` 为低频校准图、不进默认生成路径；如需统一去水印，按本流程批量跑一遍即可。
