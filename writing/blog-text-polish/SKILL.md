# 博客文字纠错 · 润色 · 去除 AI 味

一站式写作优化工具，帮你写出更像"人写"的文章。

## 功能

- **文字纠错**：错别字、语病、标点误用、搭配不当、重复啰嗦
- **句式润色**：提升可读性、节奏感、表达力度，适配博客/公众号阅读场景
- **去除 AI 味**：检测并改写 AI 写作常见模式，让文字更像真人产出
- **前后对比**：输出修改清单 + 修改前/后对比 + 完整润色版本
- **AI 指数评估**（可选）：润色前后各给出一个 AI 倾向评分，量化改善效果

## 使用方法

### 方式一：完整文章优化

```
请帮我润色这篇文章：
[粘贴整篇文章]
```

### 方式二：指定段落优化

```
请帮我改一下这段，感觉太 AI 了：
[粘贴具体段落]
```

### 方式三：只纠错/只去 AI 味

```
请帮我检查这篇文章的错别字和语病：
[粘贴内容]
```

```
请帮我去除这篇文章的 AI 味，保持原意和风格：
[粘贴内容]
```

## 执行步骤

当用户提供文章内容时，按以下流程执行：

### Step 1：接收与确认

- 确认输入来源：用户粘贴文本 / 用户粘贴 URL（用 WebFetch 获取）/ 用户上传文件
- 如果内容过长（>5000 字），告知用户将分段处理，每段单独润色
- 确认用户需求：**全面优化**（纠错+润色+去AI味）/ **仅纠错** / **仅去AI味** / **仅润色**

默认行为：**全面优化**。

### Step 2：文字纠错检查

逐句扫描以下问题类型：

| 问题类型 | 具体检查项 | 示例 |
|---------|-----------|------|
| **错别字** | 形近字、音近字常见错误 | "在" vs "再"、"的/地/得"误用、"即" vs "既" |
| **语病** | 主谓搭配、动宾搭配、成分残缺、句式杂糅 | "通过...使..."缺少主语、重复否定 |
| **标点误用** | 引号/书名号不配对、顿号逗号混用、省略号不规范 | 英文逗号混入中文文本、连续使用多个感叹号 |
| **重复啰嗦** | 同义反复、无意义修饰、车轱辘话 | "凯旋归来"、"亲眼目睹"、"十分非常" |
| **数字/单位** | 数字格式不统一、单位错误 | "100 米" vs "一百米"混用、缺失量词 |

输出格式：

```
## 📝 纠错清单

| 序号 | 原文 | 问题 | 建议修改 |
|------|------|------|---------|
| 1 | 凯旋归来 | 语义重复 | 凯旋 |
| 2 | 即管放心 | 错别字 | 尽管放心 |
```

> **说明**：纠错**必须逐条列出**，不可笼统描述。每条修改都要给出明确的"原文 → 修改建议"。

### Step 3：句式润色

在纠错基础上，对表达进行优化：

**优化原则：**
1. **长短句交错**：避免连续短句或连续长句，适当合并/拆分
2. **去掉冗余修饰**：删掉"非常"、"十分"、"真的"、"确实"等无增益副词
3. **增强画面感**：抽象描述 → 具体场景/例子/数据
4. **优化开头结尾**：开头抓人、结尾有力
5. **读者意识**：将"笔者认为/我们认为"改为自然的观点表达
6. **控制段落长度**：公众号/博客段落宜 3-5 句，过长则分

**输出格式：**

```
## ✨ 润色建议

**段落一**
- 原文：...（此处省略原文节选）
- 润色：...
- 理由：...

**段落二**
...
```

### Step 4：去除 AI 味（核心）

这是本 skill 的核心竞争力。AI 生成文本有其特殊的"气味"，逐一检测并改写：

#### 4.1 AI 味检测清单

对全文逐项检查，每项给出"✓ 通过"或"✗ 需改写"：

| 维度 | AI 味特征 | 改写方向 |
|------|----------|---------|
| **模板化序列词** | "首先/其次/再者/最后"、"第一/第二/第三" | 去掉序列词，用自然衔接 |
| **过度过渡** | "值得注意的是"、"不可忽视的是"、"从...角度看"、"与此同时" | 替换为更随意的承接，或直接去掉 |
| **虚伪平衡** | "虽然...但是..."、"既有优点也有缺点"、"一方面...另一方面..."（每件事都正反说） | 亮明立场，不需要事事都平衡 |
| **排比模板** | "不仅是...更是..."、"不仅仅是...更重要的还在于..." | 去掉模板化结构，直接说观点 |
| **万能总结** | "总的来说"、"综上所述"、"总而言之"、"以上就是关于..." | 换用更有力的收尾方式 |
| **em dash 滥用** | 密集使用"—"做解释或转折（一段出现超过 2 个） | 替换为中文常见标点，或拆分句子 |
| **数字列举癖** | "三个方面"、"五大要点"、"三个关键步骤" | 去掉数字标签，自然叙述 |
| **绝对中立** | 每个观点都"既有正面也有负面" | 允许有态度、有偏向、有情绪 |
| **缺少细节** | 通篇抽象概念，没有具体人名/地名/时间/数字/案例 | 提示用户补充细节，或从上下文推断合理细节 |
| **完美结构** | 每段都有"论点→论据→小结"，结构过于工整 | 打破规律性，有些段落可以突兀/跳跃 |

#### 4.2 改写优先级

```
高优先级（必须改）：
  ① 模板化序列词（"首先…其次…最后"）
  ② 万能总结（"总的来说/综上所述"）
  ③ 虚伪平衡（事事正反说）
  ④ 数字列举癖（"三方面/五点"）

中优先级（建议改）：
  ⑤ 过度过渡（"值得注意的是/不可忽视的是"）
  ⑥ 排比模板（"不仅是…更是…"）
  ⑦ em dash 滥用
  ⑧ 缺少细节

低优先级（有则改之）：
  ⑨ 绝对中立
  ⑩ 完美结构
```

#### 4.3 改写输出格式

```
## 🎯 AI 味检测与改写

| 维度 | 原文节选 | 改写后 | 说明 |
|------|---------|--------|------|
| 模板化序列词 | "首先，AI 技术正在..." | "AI 技术正在..." | 去掉"首先"，直接从内容开始 |
| 过度过渡 | "值得注意的是，这一趋势..." | "这一趋势..." | "值得注意的是"无信息量，直接删除 |
| 虚伪平衡 | "AI 既带来了效率提升，也存在一定风险" | 根据上下文保留一方观点 | 文章立场决定不需要各打五十大板 |
| 万能总结 | "综上所述，我们应当..." | 换用具体号召或开放问题 | 用更有力的句子收尾 |
```

> 关键原则：**保留原作者的写作风格和观点立场**，只去除"机器味"，不改变个人风格。

### 润色建议

## 注意事项

1. **不改变原意**：润色不等于重写，保持作者的核心观点、语气和风格
2. **尊重风格差异**：有些人写作就是活泼的，有些人就是严谨的，不要强行统一风格
3. **不过度修改**：如果原文已经很好了，如实告知"该段质量较高，仅做轻微调整"或无需修改
4. **提示用户补充**：如果发现某段因缺少具体细节导致内容空泛，提示用户补充真实案例/数据，而非凭空编造
5. **AI 检测参考**：本 skill 的 AI 味判断基于文本特征分析，是统计意义上的倾向判断，并非绝对判定
6. **兼容现有 skill**：与 `ai-article-detector` 互补——检测 skill 告诉你"有没有 AI 味"，本 skill 告诉你"怎么去掉 AI 味"

## 应用场景

- 博客文章发布前最终校验
- 公众号文章 AI 辅助写作后的人工化润色
- 翻译稿件的中文化润色（去除翻译腔）
- 日常邮件/报告/公告的快速润色
